# 1. 설문 신뢰도 평가 시스템 아키텍처

## 🔷 Frontend (React)
- 설문 생성 페이지
- 설문 미리보기 페이지
- 사전 문항 평가 결과 페이지
- 설문 응답 페이지
- 응답 신뢰도 결과 페이지
- 응답 필터링 / 사후 평가 페이지

## 🔷 Backend (FastAPI)
- survey 생성 / 수정 / 조회 API
- 사전 문항 평가 API
- 설문 배포 API
- 응답 제출 API
- response JSON 생성 로직
- 응답 신뢰도 평가 API
- 사후 문항 평가 API
- 결과 저장 및 조회 API

## 🔷 Database
- survey 저장
- survey item 저장
- response 저장
- response reliability result 저장
- post-hoc item evaluation 저장

## 🔷 Analysis Engine
- pipeline.py → 사전 문항 평가
- response_pipeline.py → 응답 신뢰도 평가
- scoring/* → 점수 계산
- rules/* → rule 기반 탐지
- llm/* → LLM 평가 및 rewrite

## 🔷 배포 흐름
- 설문 생성
- 사전 문항 평가
- 설문 수정 / 배포
- 응답 수집
- 응답 신뢰도 계산
- 낮은 품질 응답 필터링
- 사후 문항 평가

# 2. 전체 데이터 흐름

## A. 설문 생성 단계

Frontend
- 제목, 설명, 문항 입력
- reverse / trap / consistency_group 설정

Backend
- survey JSON 생성
- DB 저장

---

## B. 사전 문항 평가 단계

Frontend
- "문항 평가 실행" 버튼 클릭

Backend
- analyze_survey(survey) 실행

Frontend
- 점수 / 문제점 / 추천 수정안 표시
- 수정 반영 가능

---

## C. 설문 배포 단계

Backend
- share token 생성
- 링크 생성
- QR 코드 생성

---

## D. 응답 수집 단계

Frontend
- 응답값 수집
- 이벤트 로그 수집
  - response_time
  - change_count
  - revisit_count
  - focus_loss_count
  - tab_switch_count

Backend
- raw 응답 수신
- response JSON 생성
- analyze_response_reliability 실행
- 결과 저장

---

## E. 결과 확인 단계

Frontend
- 응답 점수 표시
- flags 표시
- 필터링
- 사후 평가 결과 표시

# 4. 백엔드 (FastAPI 역할)

- 응답 제출
- response JSON 생성

## response_eval.py
- 응답 신뢰도 조회
- 필터링

## posthoc.py
- 사후 문항 평가
