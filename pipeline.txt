[파일명] pipeline.py

1. 역할
- 문항 사전 신뢰도 평가의 전체 오케스트레이션 담당
- rule 기반 분석 + LLM 평가 + 점수 결합 수행

--------------------------------------------------

2. 전체 흐름

입력: survey (문항 리스트 포함 JSON)

각 item에 대해:
1) rule 기반 분석
2) LLM 보조 rule 분석 (옵션)
3) rule 결과 병합
4) feature_counts 재계산
5) penalty 계산
6) rule score 계산
7) LLM 점수 계산
8) rule + LLM 점수 결합
9) 최종 점수 계산
10) 필요 시 문장 수정 제안

→ item 결과 생성

→ 전체 survey summary 계산

출력: survey 전체 결과 JSON

--------------------------------------------------

3. 주요 함수

(1) merge_rule_analysis()
- exact rule 결과 + LLM assist 결과 병합
- 중복 제거 (feature + term 기준)

(2) rebuild_feature_counts_from_rule_analysis()
- 병합된 rule_analysis 기준으로 feature_counts 재계산

(3) analyze_item(item)
- 핵심 함수 (문항 1개 분석)
- 전체 pipeline 수행

(4) analyze_survey(survey)
- 모든 item에 대해 analyze_item 수행
- 평균 점수 계산

--------------------------------------------------

4. 입력

analyze_item(item)

item 구조:
- item_id
- question
- option_type
- options

--------------------------------------

analyze_survey(survey)

survey 구조:
- survey_id
- title
- items (list of item)

--------------------------------------------------

5. 출력

(1) item 결과 구조

- item_id
- question
- option_type
- options

- rule_analysis
- feature_counts

- penalty_details
- raw_penalties
- rule_scores

- llm_scores
- final_scores

- overall_pre_score
- overall_pre_score_100

- issue_count
- has_issue

- suggestion (문장 수정)

- exact_rule_analysis
- assist_rule_analysis

(보조 카테고리 존재 시 추가)
- auxiliary_rule_analysis
- auxiliary_feature_counts
- auxiliary_penalty_details
- auxiliary_raw_penalties
- auxiliary_rule_scores
- auxiliary_llm_scores

--------------------------------------

(2) survey 결과 구조

- survey_id
- title
- item_count

- summary
    - avg_overall_pre_score
    - avg_overall_pre_score_100
    - avg_final_scores
    - avg_auxiliary_llm_scores (옵션)

- items (각 item 결과 리스트)

--------------------------------------------------

6. 핵심 변수

- PRIMARY_CATEGORIES
- AUXILIARY_CATEGORIES
- ALL_CATEGORIES

- rule_analysis
- feature_counts

- penalty_details
- raw_penalties
- rule_scores

- llm_scores
- final_scores

--------------------------------------------------

7. 점수 계산 흐름

feature_counts
→ raw_penalties
→ rule_scores
→ LLM scores
→ final_scores (rule + llm 결합)
→ overall_pre_score

--------------------------------------------------

8. 다른 파일과의 연결

입력:
- main.py에서 호출됨

사용:
- rules.matcher (feature 추출)
- scoring.rule_scoring (penalty 계산)
- scoring.score_combiner (점수 결합)
- llm.eval_evaluator (LLM 점수)
- llm.rule_assist_evaluator (LLM rule 보조)
- llm.rewrite (문장 수정)

향후 연결:
- survey_augmenter.py에서 호출 예정
- response_pipeline.py와는 독립

--------------------------------------------------

9. 주의사항 (중요)

- 이 파일은 "문항 사전 평가 전용"
- 응답 신뢰도 로직 절대 넣지 말 것
- pipeline 구조는 유지하는 것이 좋음

--------------------------------------------------

10. 핵심 한 줄 요약

문항 → feature 추출 → penalty → 점수 → LLM 보정 → 최종 문항 신뢰도
