[서비스 구현 단계]

1. 설문 생성 UI (프론트)
- 문항 추가/삭제
- 옵션 타입 선택
- 메타 설정 (reverse, trap 등)
- survey JSON 생성

2. 설문 저장 및 관리 (백엔드)
- survey JSON DB 저장
- survey_id 관리

3. 사전 문항 평가 기능
- analyze_survey(survey) 호출
- 문항 점수 및 문제 표시
- rewrite suggestion 적용 기능

4. 설문 배포 기능
- 링크 생성
- QR 코드 생성

5. 응답 수집 기능 (프론트)
- answer_value 수집
- 이벤트 로그 수집
    - response_time
    - tab_switch
    - focus_loss
    - revisit

6. response JSON 생성 (백엔드 핵심)
- raw 데이터 → 구조화
- response_id 생성
- participant_id 생성
- question_log 구성
- survey_log 구성

7. 응답 신뢰도 평가 기능
- analyze_response_reliability 실행
- 결과 저장

8. 결과 시각화 (프론트)
- 응답 점수 표시
- flags 표시
- 문제 feature 표시

9. 필터링 기능
- 낮은 신뢰도 응답 제거
- 재계산 기능
